# Product Documents

This directory stores investigation notes, requirements, and design documents for the image-to-wood-constructor business idea.

## Structure

- `investigations/`: market research, business models, experiments, and validation notes.
- `requirements/`: product requirements, MVP scope, acceptance criteria, and roadmap requirements.
- `design/`: UX flows, system design, manufacturing design, and technical architecture.
- `manufacturing/`: machine capability schema, tool database, AI skills, and RAG/manual ingestion plan.

## Current Idea

Build a marketplace where users can buy ready-made crafted items and generate supported custom products with credits. The MVP catalog includes toys, constructors, decorations, and night lights. Custom generation starts with night lights and 2D laser-cut items, while admins manage catalog items, generated outputs, and production review.

## Key MVP Documents

- `requirements/mvp-marketplace-requirements.md`: marketplace MVP scope, roles, flows, and acceptance criteria.
- `design/mvp-marketplace-system-design.md`: proposed architecture, data model, generation pipelines, admin review, and implementation phases.
- `manufacturing/tools.json`: first-pass structured capability database for available and quoted machines.
- `manufacturing/ai-skills.md`: AI workflows for tool selection, manufacturability, file preparation, quoting, and operator checklists.
- `manufacturing/rag-manuals.md`: manual/RAG source plan and exact-model gaps.
